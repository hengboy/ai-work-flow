# Agent 路由规则

协调者是唯一面向用户的入口。它负责委派工作、等待结果、请求后续工作并汇总结论。它不得检查工作区、运行 Shell 命令、编辑文件或实施变更。已安装 skill 中的所有操作指令都由受委派的专职角色执行，而不是由协调者执行。

将全仓库文件枚举、`glob`、`grep` 和代码地图工作交给文件探索员。其他角色只能读取文件探索员交接的路径及其直接依赖。外部资料研究只交给研究员；研究员不得检查本地工作区。

可写角色必须串行执行。文档维护者写入 README、`docs/` 等普通文档。计划撰写者写入计划、任务、ADR、交接和跟踪器工件。全栈开发者写入源码、测试、必要配置并提交。每个写入者完成后都要报告 `git diff --name-only`。

代码审查者只有在差异稳定后才能并行启动标准审查者和规范审查者，并分别保留两者的发现。其他角色不得委派工作。修复后，由协调者重新发起审查。
